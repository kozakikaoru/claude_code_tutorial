第 1.3 章 intro

「自動の度合い」を握る
権限モード

ファイルを書き換えられるのは怖いけれど、毎回ぜんぶ確認するのも面倒——その塩梅を握るのが 権限モードPermission mode編集やコマンド実行を、どこまで人に確認するかを決める設定。モデルの賢さは変えず「実行前に止めるか/止めないか」だけを切り替えます。。アクセルの踏み込みを決めるダイヤルだと思ってください。

慎重にいきたいときは確認多めに、慣れてきたら自動多めに。この章では 4 つのモードと切り替え方、そして会話のひとまとまりである セッションSession「会話履歴 + 対象フォルダー + 加えた変更」を 1 単位にしたもの。閉じても残り、あとから再開できます。の再開を学びます。これが分かれば、安心して仕事を任せられます。

これは「賢さ」ではなく「自動の度合い」の設定です。 どのモードでも能力は同じ。変わるのは、実行前にあなたへ確認するかどうかだけ。次のページで 4 種類を見ていきます。

§14 modes

4 つのモード、ひと目で

左ほど安全(確認多め)、下にいくほど自動。まずは default から。bypassPermissions は原則使いません。

モード何が起きるかいつ使うか
default 読み取りは許可。編集・実行は その都度確認 標準。まずはこれ
plan 編集も実行もしない。計画だけ立てて見せる 大きめの作業前に方針レビュー
acceptEdits 編集の確認を スキップ。実行は確認 方針が固まり速く進めたい
bypassPermissions 確認なしの 全権委任(危険) 原則使わない。隔離環境のみ

安全側の default と plan で十分に戦えます。 acceptEdits は「変更を後で必ず差分で見る」習慣とセットで。

§2switch

モードを切り替える

いまの表示 · ターミナル版 デスクトップ版

対話中に Shift+Tab を押すたびモードが 巡回。現在モードは画面下部の ステータスラインStatus lineターミナル下部に出る現在の状態表示。モード・モデル・作業フォルダーがひと目で分かります。 に出ます。

画面の トグルやメニュー から選びます。Shift+Tab の巡回も使えますが、UI スイッチで「計画だけ」「編集を自動承認」を明示的に選べます。

zsh — claude モード表示 CLIApp 
# Shift+Tab を押すたびモードが巡回  ⎿ mode plan （解析のみ・編集も実行もしない）   status: plan · Sonnet · ~/projects/my-app> # 画面上のトグル / メニューでモードを選ぶモード  プラン （解析のみ・編集も実行もしない）   現在: プラン · Sonnet · my-app

迷ったら、まず plan に。 計画を読んで納得してから default / acceptEdits で実行——この往復が基本形です。

§3session

続きから再開する

セッションSession「会話履歴 + 対象フォルダー + 加えた変更」を 1 単位として持つ作業のまとまり。 は、何を話し・どこで・どこを変えたかをセットで記録します。うれしいのは あとから再開 できること。「昨日の続き」を説明し直さずに戻れます。

zsh — claude セッション一覧 CLIApp 
# 直前のセッションをそのまま再開$ claude -c # 一覧から選んで再開$ claude --resume  ⎿ 過去のセッションから選ぶと会話ごと復元# サイドバーに過去のセッションが並ぶ● 認証まわりのリファクタ   my-app · 今日○ テストの追加          my-app · 昨日  → クリックで会話履歴ごと復元される

セッションは消えません。 閉じても会話・フォルダー・変更はひとまとまりで残り、呼び戻せます。だから「試してダメなら閉じる」を気軽に。

§4safety

慣れるまでは確認多めで

結論はシンプル。慣れるまでは default か plan を中心に。この 2 つなら編集・実行の前に必ずあなたの目を通せ、Claude Code の挙動への勘所も早く身につきます。

bypassPermissions を安易に使わないでください。 確認をいっさい省いて全権委任します。ファイル削除や危険なコマンドも止める間もなく実行され得ます。初級のうちは存在を知っておくだけで十分です。

権限モードで「自動の度合い」を、セッションで「作業のまとまり」を扱えるようになりました。次章では毎回自動で読まれる取扱説明書 CLAUDE.mdCLAUDE.md各セッションの最初に自動で読まれるプロジェクトの取扱説明書。「うちのやり方」を最初から共有できます。 を用意します。

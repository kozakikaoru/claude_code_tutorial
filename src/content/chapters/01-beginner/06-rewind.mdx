ここまでで、自然言語で指示を投げ、差分を確認し、よく使う操作まで一通りなぞってきました。それでもまだ、こう感じている人は多いはずです——「変なことをして、プロジェクトを壊したらどうしよう」。

この章は、その不安をほどくためにあります。結論から言うと、**Claude Code は失敗を戻せる世界**です。直前の変更が気に入らなければ、なかったことにできる。それが分かれば、初めての操作も思いきって試せます。

<Callout>
  この章のゴールは、技術というより気持ちの面にあります。「**戻せると分かっているから、安心して試せる**」——その状態になることが、初級でいちばん大切なことのひとつです。
</Callout>

## §1 巻き戻しとは

<Term id="rewind">巻き戻し</Term>は、エージェントが行った**直前の変更をなかったことに戻す**機能です。たとえば「リファクタしてほしい」と頼んだら思っていた形と違った、というとき。手作業で元に戻す必要はありません。巻き戻しを呼べば、変更前の状態にすっと戻せます。

大事なのは、これが**やり直しの安全網**になっているという点です。「失敗したら戻せばいい」と思えれば、最初の一歩のハードルはぐっと下がります。気軽に試して、違ったら戻す——この身軽さこそが、Claude Code を使ううえでの基本姿勢です。

<ModeBlock mode="terminal">
### ターミナルでは、こう戻す

ターミナル版では、二つの呼び出し方があります。ひとつは<Term id="slash-command">スラッシュコマンド</Term>の `/rewind`。もうひとつは、**Esc キーを 2 回**続けて押す方法です。どちらでも戻り先の一覧が開き、「どこまで戻すか」を選べます。

<CodeBlock title="zsh — claude" badge="操作">
<span class="line"><span class="sx-com"># 直前の変更を巻き戻す（スラッシュコマンド）</span></span>
<span class="line"><span class="prompt cy">&gt;</span> <span class="sx-key">/rewind</span></span>
<span class="line"> </span>
<span class="line"><span class="sx-com"># または Esc を 2 回押す（Esc Esc）でも同じ画面が開く</span></span>
<span class="line"><span class="sx-dim">  ⎿ </span><span class="sx-fn">戻り先を選択</span> <span class="sx-flag">↑↓ で選んで Enter</span></span>
<span class="line"><span class="sx-dim">  ⎿ </span><span class="sx-str">● いま</span> <span class="sx-dim">（最新の状態）</span></span>

### デスクトップでは、こう戻す

デスクトップ版では、戻り先を**視覚的に選んで**巻き戻します。これまでの変更の区切りが一覧で並んでいるので、戻したいところをクリックするだけ。コマンドを覚えなくても、画面を見ながら直感的に操作できます。

<CodeBlock title="巻き戻しパネル" badge="操作">
<span class="line"><span class="sx-com"># 変更の区切りが一覧で並ぶ。戻りたいところを選ぶだけ</span></span>
<span class="line"><span class="prompt">いま</span>      <span class="sx-str">最新の状態</span>  <span class="sx-dim">●</span></span>
<span class="line"><span class="sx-dim">        </span><span class="sx-flag">src/auth.ts を編集する前</span>  <span class="sx-str">［ここに戻す］</span></span>
<span class="line"><span class="sx-dim">        </span><span class="sx-flag">テストを追加する前</span>  <span class="sx-str">［ここに戻す］</span></span>
<span class="line"><span class="sx-dim">        変更前後は差分ビューで見比べられます</span> <span class="caret" aria-hidden="true"></span></span>
</CodeBlock>
</ModeBlock>

どちらのモードでも、やっていることは同じです。**戻り先を選んで、そこまで戻す**。入り口がコマンドかクリックかの違いだけです。

## §2 チェックポイントという「戻り先」

巻き戻しを呼ぶと戻り先がいくつか出てきますが、この一つひとつが<Term id="checkpoint">チェックポイント</Term>です。チェックポイントとは、**変更の区切り点**のこと。エージェントが作業を進める節目で自動的に作られていきます。

ゲームのセーブポイントを思い浮かべると分かりやすいでしょう。作業の区切りごとにセーブが残り、「あのセーブまで戻る」を選べる——巻き戻しは、その戻り先としてチェックポイントを使います。自分で「ここを保存」と意識しなくても、節目は勝手に記録されているのです。

ここで二つ、押さえておきたいことがあります。

- **どこまで戻れるか** — 戻り先は直前の一手だけではありません。いくつか前の区切りまでさかのぼって選べます。「二つ前のあの状態に戻したい」も可能です。
- **戻すと何が起きるか** — 選んだチェックポイント以降の変更は取り消され、その時点の状態に戻ります。だからこそ、戻す前に「どこまで取り消されるのか」を一度確認しておくと安心です。

<Callout type="warn">
  <b>戻すと、その先の変更は消えます。</b> 巻き戻しは「やり直し」ではなく「巻き戻し」です。あるチェックポイントまで戻すと、それより後にやった変更はなくなります。残しておきたい変更があるなら、戻す前にどこまで取り消されるかを確かめてください。とはいえ、ここまで読んで身構える必要はありません。**迷ったら、まず戻り先の一覧を眺めるだけ**でも十分です。
</Callout>

## §3 だから、自動承認も怖くない

前章までで<Term id="permission-mode">権限モード</Term>に触れました。なかでも `acceptEdits`（編集を自動承認するモード）は、エージェントの編集をいちいち確認せず、どんどん進めてくれる便利なモードです。一方で、「確認なしで変更されるなんて、ちょっと怖い」と感じた人もいるかもしれません。

ここで巻き戻しが効いてきます。**自動承認でどんどん変更が進んでも、気に入らなければチェックポイントまで戻せばいい**。つまり、自動承認の身軽さと、巻き戻しの安全網は、セットで使うことで本領を発揮します。前に進む速さと、いつでも戻れる安心。この二つがそろって初めて、安心してアクセルを踏めるのです。

<Callout>
  <b>Claude Code は、Undo（取り消し）のある世界です。</b> 文書作成ソフトで「元に戻す」が当たり前にあるように、Claude Code にも巻き戻しがあります。だから、おそるおそる進める必要はありません。気軽に試して、違ったら戻す——その繰り返しのなかで、いちばん早く上達していきます。
</Callout>

巻き戻しがあると分かれば、これまでの章で学んだことを実際に試すハードルは、もうほとんどありません。「壊したらどうしよう」ではなく、「とりあえずやってみよう、ダメなら戻せる」。その気持ちで進めるのが、初級を抜けるいちばんの近道です。

ここまでで、初級パートの基本はひととおりそろいました。起動し、対話し、操作し、そして失敗しても戻せる——この土台の上に、次のパートからは Claude Code を**自分仕様に育てる**話へと進んでいきます。

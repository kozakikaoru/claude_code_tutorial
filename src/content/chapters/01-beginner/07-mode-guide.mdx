初級もこれが最後の章です。ここまでで、インストール・最初の対話・差分の確認・よく使う操作と、ひととおりの基礎は手に入りました。この章のゴールは、それらを**自分が使っているモードの画面**にきちんと結びつけること。「あの操作は、この画面のどこにあるのか」が分かれば、初級は卒業です。

この章は全体が**モード別の二本立て**です。右上のトグルで自分の環境を選ぶと、その歩き方だけが表示されます。最後の比較表だけは両モード共通で出します。

<ModeBlock mode="terminal">
<Callout>
  <b>ターミナル版を選んでいます。</b> ここからは <Term id="cli">CLI</Term> の画面の見方を、起動直後から順にたどります。デスクトップ版の歩き方を読みたいときは、右上のトグルを切り替えてください。
</Callout>

## §1 起動直後の画面を読む

プロジェクトのフォルダーで `claude` を起動すると、対話用の画面に切り替わります。最初に押さえる場所は二つだけです。**入力欄**（プロンプト `>`）と、その下に流れていく**出力**です。

<CodeBlock title="zsh — claude" badge="CLI">
<span class="line"><span class="prompt">$</span> <span class="sx-key">claude</span></span>
<span class="line"> </span>
<span class="line"><span class="sx-dim">  ようこそ。やりたいことを日本語で入力してください。</span></span>
<span class="line"> </span>
<span class="line"><span class="prompt cy">&gt;</span> <span class="caret" aria-hidden="true"></span></span>
<span class="line"> </span>
<span class="line"><span class="sx-dim">  ~/projects/my-app  ·  default  ·  Sonnet</span></span>
</CodeBlock>

`>` の行があなたの**入力欄**です。ここに指示を打って Enter を押すと、エージェントが動き出します。いちばん下に出ている薄い色の一行が<Term id="status-line">ステータスライン</Term>です。次の §2 でじっくり読みます。

## §2 ステータスラインを読む — モードとモデル

<Term id="status-line">ステータスライン</Term>は、いまの状態をひと目で確認できる場所です。初級のうちは、ここに出る二つだけ読めれば十分です。

- **権限モード** — 上の例の `default` の部分です。いまどこまで自動で実行してよい状態か（`default` / `plan` / `acceptEdits` など）を示します。<Term id="plan-mode">プランモード</Term>に入っていれば、ここが `plan` に変わります。
- **モデル** — 上の例の `Sonnet` の部分です。いま動いている<Term id="models">モデル</Term>（Opus / Sonnet / Haiku）が分かります。

「思ったより勝手に編集された」「やけに慎重だ」と感じたときは、まずここを見ます。多くの場合、原因は権限モードの取り違えです。前章で覚えた `Shift+Tab` でモードを巡回させると、この表示がそれに合わせて変わるのを確認できます。

<Callout>
  ステータスラインに出る項目や並びは設定や版によって少し変わります。場所と意味（モード・モデルがここで分かる）さえ掴んでおけば十分です。細かなカスタマイズは上級で扱います。
</Callout>

## §3 入力欄とストリーミング出力

指示を打つと、エージェントの作業が**上から順に流れて**表示されます。読み取り・編集・コマンド実行が、起きた順に一行ずつ追記されていく——これがターミナル版の基本の見え方です。

<CodeBlock title="zsh — claude" badge="CLI">
<span class="line"><span class="prompt cy">&gt;</span> <span class="sx-str">"README に使い方の節を追記して"</span></span>
<span class="line"> </span>
<span class="line"><span class="sx-dim">  ⎿ </span><span class="sx-fn">read</span> <span class="sx-flag">README.md</span> <span class="sx-dim">(84 行)</span></span>
<span class="line"><span class="sx-dim">  ⎿ </span><span class="sx-fn">edit</span> <span class="sx-flag">README.md</span> <span class="sx-str">+22 -0</span></span>
<span class="line"><span class="prompt cy">&gt;</span> <span class="caret" aria-hidden="true"></span></span>
</CodeBlock>

長い出力の途中で止めたくなったら `Esc`、行き過ぎたら `Esc` 2 回で<Term id="rewind">巻き戻し</Term>——どちらも前章で覚えたとおりです。差分はこの流れの中にそのまま表示されるので、**読んで、納得してから取り込む**のが基本姿勢でした。

## §4 この先に出てくるもの（予告）

ターミナル版には、初級ではまだ使わない入口がいくつもあります。いまは「あるんだな」と知っておくだけで十分です。

- **起動時のフラグ** — `claude` の後ろに付ける細かなオプションがあります。たとえばモデルを指定する `--model` のように、起動のしかたを変えられます。詳しい使い方は上級・付録でまとめて扱います。
- **他のコマンドへ繋ぐ（パイプ）** — ターミナル版の出力は、別のコマンドに渡して組み合わせられます。「ログを Claude に渡して要約させる」といった連携の入口で、自動化の章で本格的に登場します。

<Callout type="warn">
  フラグやパイプは便利ですが、初級でいきなり手を出す必要はありません。まずは `claude` を起動して、日本語で指示する——この基本だけで初級は十分です。ここでは存在を頭の隅に置いておけば大丈夫です。
</Callout>
</ModeBlock>

<ModeBlock mode="desktop">
<Callout>
  <b>デスクトップアプリ版を選んでいます。</b> ここからは <Term id="desktop">デスクトップアプリ</Term>の画面の見方を、メイン画面の構成から順にたどります。ターミナル版の歩き方を読みたいときは、右上のトグルを切り替えてください。
</Callout>

## §1 3 つのタブを把握する

アプリを起動すると、上部に **Chat / <Term id="cowork">Cowork</Term> / Code** の 3 つのタブが並んだメイン画面が開きます。まずはこの三択の役割を掴むことが、デスクトップ版を歩く第一歩です。

- **Chat** — 軽い相談や質問をする画面。「これってどういう意味?」のような短いやりとり向きです。
- **Cowork** — 腰を据えた共同作業の画面。エージェントに腰の重いタスクを任せ、進み具合を見ながら一緒に進めます。
- **Code** — コードそのものに向き合う画面。ファイルを開いて、変更を確認しながら作業します。

最初は **Chat で質問してみる → Cowork で実際のタスクを任せる**、という行き来ができれば十分です。タブはいつでも切り替えられます。

## §2 ファイルツリーで「どこを触るか」を見る

画面の横には<Term id="file-tree">ファイルツリー</Term>が出ます。プロジェクトのフォルダーとファイルが階層で並んだ、おなじみの一覧です。

ここは二つの役割を持ちます。ひとつは、いまどんな構成のプロジェクトを開いているかを**俯瞰**すること。もうひとつは、見てほしいファイルを選んで**会話の文脈に加える**ことです。ターミナル版の <Term id="at-mention">@メンション</Term> に当たる操作を、ここではクリックで行えます。

<Callout>
  「このファイルを見て直して」と頼みたいとき、ファイルツリーから対象を選んでおくと、エージェントが迷わず読みにいけます。どのファイルが関係するか自分で分かっているときに便利です。
</Callout>

## §3 差分ビューで変更を確認する

エージェントがコードを書き換えると、変更は<Term id="diff-view">差分ビュー</Term>に表示されます。変更前と変更後を**横に並べて見比べられる**のが、デスクトップ版のいちばんの強みです。

<CodeBlock title="差分ビュー" badge="例">
<span class="line"><span class="prompt cy">あなた</span>  <span class="sx-str">README に使い方の節を追記して</span></span>
<span class="line"> </span>
<span class="line"><span class="prompt">Claude</span>  <span class="sx-fn">読み取り</span> <span class="sx-flag">README.md</span></span>
<span class="line"><span class="sx-dim">        </span><span class="sx-fn">編集</span>     <span class="sx-flag">README.md</span>  <span class="sx-str">+22 −0</span></span>
<span class="line"><span class="sx-dim">        差分ビューで変更を確認してください</span> <span class="caret" aria-hidden="true"></span></span>
</CodeBlock>

ここでも基本姿勢は同じです。提案された差分を**読んで、納得してから取り込む**。緑（追加）と赤（削除）で色分けされた変更に目を通し、問題なければ承認します。これが、安心して任せるための土台でした。

## §4 この先に出てくるもの（予告）

デスクトップ版には、初級ではまだ使わない便利な仕組みがいくつもあります。いまは「あるんだな」と知っておくだけで十分です。

- **複数セッションの並列** — 別々のタスクを同時に走らせ、進み具合を画面で見比べられます。「片方を待っている間にもう片方を進める」が一画面でできます。<Term id="session">セッション</Term>を複数持てる、という前章の話の発展形です。
- **定期タスク** — 決まったタイミングで自動的に処理を走らせる仕組みもあります。自動化の入口で、後の章で本格的に扱います。

<Callout type="warn">
  これらは強力ですが、初級でいきなり使う必要はありません。まずは Chat と Cowork で日本語の指示を投げ、差分ビューで確認する——この基本だけで初級は十分です。ここでは存在を頭の隅に置いておけば大丈夫です。
</Callout>
</ModeBlock>

## §5 ふたつのモードを見比べる

ここまで自分のモードの歩き方を見てきました。最後に、ターミナル版とデスクトップ版を並べて確認しておきましょう。下の表は両モード共通です。

| 観点 | ターミナル版 | デスクトップ版 |
|---|---|---|
| 向いている人 | エンジニア、自動化したい人、Linux で使う人 | 初心者・非エンジニア・PM、視覚的に確認したい人 |
| 主な操作 | 入力欄に打って、流れる出力を読む | 3 タブを切り替え、ファイルツリーと差分ビューで進める |
| 強み | フラグでの細かな制御、他コマンドとの連携（パイプ）、自動化に強い | 視覚的な差分確認、複数セッションの並列、定期タスク |

向き不向きはありますが、どちらかが上位というわけではありません。中身の<Term id="models">モデル</Term>も能力もまったく同じで、入り口が違うだけです。だから——

<Callout>
  <b>両方使ってもかまいません。同じ土台の上に立っています。</b> 普段はターミナル版で素早く回し、変更が大きいときだけデスクトップ版で差分をじっくり見る、といった使い分けも自然にできます。<Term id="claude-md">CLAUDE.md</Term> や <Term id="mcp">MCP</Term> などの設定はどちらのモードでも共通で効くので、入り口を変えても積み上げた設定は無駄になりません。
</Callout>

これで初級は修了です。<Term id="claude-code">Claude Code</Term> を起動し、自然言語で指示を投げ、差分を確認し、自分の画面のどこに何があるかまで掴めました。ここからが本番です。第2部（中級)では、Claude を「優秀な新卒部下」として働きやすくする**環境づくり**——規約を書く CLAUDE.md、安全装置の権限設定、そして中級の山である<Term id="subagent">サブエージェント</Term>——へと進みます。

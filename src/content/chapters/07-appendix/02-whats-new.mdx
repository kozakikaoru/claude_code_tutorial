<Term id="claude-code">Claude Code</Term> は、本サイトのどの機能よりも速いペースで更新され続けています。新しいコマンド、新しいショートカット、課金の変更——数週間で景色が変わることも珍しくありません。だからこそ大事なのは、個々の値を丸暗記することではなく、**最新版を自分で確かめる手順を持っておく**ことです。

<Callout type="warn">
  <b>このサイトの内容は「2026-06 時点のスナップショット」です。</b> 執筆時点で確認できた情報をもとにしていますが、Claude Code の公式 What's new は**週次で更新**されます。コマンド名・ショートカット・上限値・課金などは、お使いのバージョンで実際の挙動が変わっている可能性があります。迷ったら、本ページの手順で最新の一次情報を確認してください。
</Callout>

## §1 まず一次情報を見る

更新の出どころは公式ドキュメントです。中でも次の二つを押さえておけば十分です。

- **公式ドキュメント** — 各機能の正確な仕様はここが正本です。本サイトと食い違ったら、つねに公式が優先します。
- **What's new（変更履歴）** — 何が・いつ追加/変更されたかが時系列で並びます。**週次で更新**されるので、ここを見れば「自分の知識が古くないか」をすぐ確かめられます。

ブログやまとめ記事は素早く全体像を掴むのに便利ですが、数字や仕様は一次情報で裏を取る——この順番を習慣にしてください。

## §2 手元のバージョンと使えるコマンドを確かめる

公式を見る前後で、いま動いている自分の環境も確認しておきます。やることは三つだけです。

<CodeBlock title="zsh — バージョン確認" badge="CLI">
<span class="line"><span class="sx-com"># 今入っている Claude Code のバージョンを表示</span></span>
<span class="line"><span class="prompt">$</span> claude <span class="sx-flag">--version</span></span>
</CodeBlock>

- **バージョンを見る** — 上記コマンドで、いま動いている版が分かります。What's new の記載と突き合わせれば、自分が新機能を持っているか判断できます。
- **<code class="inline">/help</code> を打つ** — セッション中に <Term id="slash-command">スラッシュコマンド</Term> の一覧が出ます。お使いの版で**実際に使えるコマンド**が確認できるので、本サイトの一覧と差分があればこれが答えです。
- **更新する** — 動作が古いと感じたら更新します。ネイティブ版は <Term id="auto-update">自動更新</Term> されますが、Homebrew / WinGet 版は手動です（付録外・初級「インストール」の章を参照）。

## §3 特に変わりやすい項目（要再確認リスト）

次の項目は版による変動が大きく、本サイトの記述が古くなりやすいところです。実務で頼る前に、最新版で必ず確認してください。

- **ショートカットと入力テク** — キー割り当てや <code class="inline">@</code> / <code class="inline">#</code> の挙動は版で変わります。<code class="inline">/help</code> と公式リファレンスで最新を確認。
- **スキルとスラッシュコマンドの統合** — <Term id="skill">スキル</Term> と <Term id="custom-slash-command">カスタムスラッシュコマンド</Term> の扱いは整理が進んでいます。`.claude/skills/` と `.claude/commands/` の関係は公式で裏取りを。
- **入れ子の階層数・控えのモデル数** — <Term id="nested-subagent">入れ子サブエージェント</Term> の最大階層や <Term id="fallback-model">fallbackModel</Term> の本数、<Term id="loop-command">/loop</Term> の継続上限などの「数」は、本サイトでは目安です。最新値は公式で確認。
- **課金** — <Term id="monthly-credit">月次クレジット</Term> をはじめ料金体系は改定されることがあります。自動化を本格運用する前に、必ず最新の課金ページを確認してください。

<Callout>
  <b>古さは「事故」ではなく「前提」です。</b> 速く動くツールを追うコツは、最新を暗記することではなく、**ここがズレているかも、と疑って一次情報に当たる癖**をつけること。この一手間が、誤った前提で時間を溶かす事故を防ぎます。
</Callout>

この付録は、本編で登場した操作を**引くためのリファレンス**です。覚える章ではなく、忘れたときに戻ってくる章だと思ってください。各表には「モード」列を設け、その操作が <Term id="cli">CLI</Term>（ターミナル版）だけのものか、デスクトップアプリ版でも同じように使えるかを示します。

<Callout>
  <b>「ターミナルのみ」でも、多くは GUI に同等の入口があります。</b> キー操作やフラグはターミナル版固有でも、デスクトップ版では同じ操作をメニューやボタンから行えることが多いです。各表のあとに、その対応を補足しています。
</Callout>

## §1 この早見表の使い方

表は 3 つに分かれています。**スラッシュコマンド**（§2）／**キーボードショートカット**（§3）／**主要 CLI フラグ**（§4）です。それぞれ独立して引けます。

- **モード列の読み方** — 「両対応」はターミナル版とデスクトップアプリ版のどちらでも使えるもの、「ターミナルのみ」は CLI 固有のものです。
- **点線の語** — <Term id="slash-command">スラッシュコマンド</Term> のように下線が引かれた語は、カーソルを合わせる（スマートフォンはタップ）と用語の説明が出ます。意味があやふやなら、まずそこを触ってください。
- **詳しい解説** — 各操作の「なぜ・いつ使うか」は本編の該当章にあります。この付録は要点だけを短くまとめています。

最後の §5 に、**版による変動についての注意**をまとめました。早見表を信頼しすぎる前に、一度目を通しておいてください。

## §2 スラッシュコマンド

入力欄で `/` から始める組み込みの命令です。ターミナル版・デスクトップアプリ版のどちらでも `/` を打つと候補が出ます。

| コマンド | 何をする | モード |
|---|---|---|
| `/help` | コマンドの一覧と使い方を表示 | 両対応 |
| `/init` | リポジトリを解析して <Term id="claude-md">CLAUDE.md</Term> を生成 | 両対応 |
| `/clear` | 会話履歴をリセット（文脈を掃除） | 両対応 |
| `/compact` | 会話を圧縮して文脈を保ったまま継続 | 両対応 |
| `/model` | 使うモデル（Opus / Sonnet / Haiku）を切り替え | 両対応 |
| `/permissions` | ツールの許可（どこまで自動実行するか）を設定 | 両対応 |
| `/agents` | <Term id="subagent">サブエージェント</Term> を作成・編集・管理 | 両対応 |
| `/hooks` | 設定済みの <Term id="hook">フック</Term> を確認 | 両対応 |
| `/review` | 変更内容のコードレビューを依頼 | 両対応 |
| `/loop` | プロンプトを一定間隔で繰り返す（ポーリング用） | ターミナルのみ |
| `/desktop` | いまのセッションをデスクトップアプリへ渡す | ターミナルのみ |
| `/exit` | セッションを終了 | 両対応 |

<Callout>
  デスクトップアプリ版では、これらの多くは**コマンドパレットやメニュー**からもボタンで実行できます。コマンド名を覚えていなくても、GUI 側の対応する場所を探せば同じことができます。
</Callout>

## §3 キーボードショートカット

ターミナル版の操作を速くするキー操作です。キーボード操作そのものは CLI のものですが、巻き戻しやファイル投入など、デスクトップアプリ版に同等の入口があるものは「両対応」としています。

| 操作 | 効果 | モード |
|---|---|---|
| `Shift+Tab` | <Term id="permission-mode">権限モード</Term> を巡回（default → 編集自動承認 → plan） | ターミナルのみ |
| `Esc` | 実行中の処理を中断 | ターミナルのみ |
| `Esc` `Esc` | 入力履歴をさかのぼる／直前の変更を <Term id="rewind">巻き戻し</Term> | 両対応 |
| `Ctrl+C` を 2 回 | セッションを終了 | ターミナルのみ |
| `Ctrl+L` | 画面をクリア（履歴は保持） | ターミナルのみ |
| `Ctrl+B` | 実行中の処理を <Term id="background-agent">バックグラウンド</Term> へ送る | ターミナルのみ |
| `Tab` | コマンドやパスを補完 | ターミナルのみ |
| `↑` `↓` | 入力履歴を巡回 | ターミナルのみ |
| `Shift+Enter` | 改行（送信せずに次の行へ） | ターミナルのみ |
| `@ファイル名` | ファイルを文脈に投入する <Term id="at-mention">@メンション</Term> | 両対応 |
| `#テキスト` | 行頭の `#` で <Term id="hash-memory">メモリに追記</Term> | 両対応 |

<Callout>
  デスクトップアプリ版では、`@` のファイル投入は <Term id="file-tree">ファイルツリー</Term> から選ぶ操作に、巻き戻しは <Term id="checkpoint">チェックポイント</Term> を選ぶ UI ボタンに対応します。キーを覚えるより画面を見て操作したい人は、GUI 側を使ってください。
</Callout>

## §4 主要 CLI フラグ

`claude` 起動時に付けるオプションです。フラグはターミナル版（CLI）固有の指定方法のため、いずれも「ターミナルのみ」です。`<...>` の部分は自分の値に置き換えます。

| フラグ | 何をする | モード |
|---|---|---|
| `-p` / `--print` | <Term id="headless">ヘッドレス</Term> 実行（1 回だけ実行して結果を返す） | ターミナルのみ |
| `--model <名前>` | 使うモデルを指定して起動 | ターミナルのみ |
| `--permission-mode <モード>` | <Term id="permission-mode">権限モード</Term> を指定して起動 | ターミナルのみ |
| `-c` / `--continue` | 直前のセッションを再開 | ターミナルのみ |
| `--resume <名前>` | 名前を付けたセッションを復元 | ターミナルのみ |
| `--output-format <形式>` | `-p` の出力形式（`text` / `json` など）を指定 | ターミナルのみ |
| `--max-turns <回数>` | 応答の往復回数に上限を設ける | ターミナルのみ |
| `--allowedTools <一覧>` | 使ってよいツールを限定する | ターミナルのみ |
| `--teleport` | Web / iOS の長時間タスクをターミナルへ引き込む | ターミナルのみ |
| `--safe-mode` | フックやプラグインの初期化をスキップして起動 | ターミナルのみ |

<Callout>
  デスクトップアプリ版にフラグはありませんが、モデル選択・権限モード・セッション復元といった**同じ設定は GUI から行えます**。自動化やパイプ合成にフラグを使いたい場合は、ターミナル版（または `-p` のヘッドレス実行）が土俵です。
</Callout>

## §5 版による変動に注意

この早見表は **2026 年 6 月時点のスナップショット** です。Claude Code は更新が速く、とくに**キーボードショートカットの割り当て・スラッシュコマンドの追加や統廃合・新しいフラグ**は版によって変わります。

<Callout type="warn">
  <b>「表と違う」と感じたら、まず最新版を確認。</b> ショートカットが効かない、コマンドが見つからない——そんなときは、ここの記載が古い可能性があります。ターミナル版なら `/help` でいまのコマンド一覧を、各操作の最新仕様は公式の更新情報（What's new）を確認してください。最新の追い方は付録 C で扱います。
</Callout>

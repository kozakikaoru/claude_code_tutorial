## §1 /btw — 会話を汚さず横に聞く

作業の途中で「ところでこのコマンドの意味は?」とちょっと聞きたくなることがあります。普通に聞くと、その質問と回答が本筋の履歴に残り、<Term id="context">コンテキスト</Term> が散らかります。

そこで <Term id="btw">/btw</Term> です。`/btw` に続けて質問すると、本流の会話を汚さずに横で答えてくれます。やり取りが履歴に積もらないので、文脈を保ったまま気軽に脱線できます。

<CodeBlock title="zsh — claude" badge="CLI">
<span class="line"><span class="sx-com"># 本筋を中断せず、横でさっと質問する</span></span>
<span class="line"><span class="prompt cy">&gt;</span> <span class="sx-key">/btw</span> <span class="sx-str">git rebase と merge の違いを一言で</span></span>
</CodeBlock>

## §2 !コマンド — 推論を挟まず即シェル実行

決まったシェルコマンドをサッと打ちたいだけなのに、毎回エージェントに頼むと推論が入ってまどろっこしい——そんなときは <Term id="bang-command">!コマンド</Term> です。

行頭に `!` を付けて入力すると、Claude の推論を挟まずに、そのままシェルコマンドが即実行されます。`!git status` や `!npm test` のように、結果だけ手早く見たいときに速いです。

<CodeBlock title="zsh — claude" badge="CLI">
<span class="line"><span class="sx-com"># 行頭の ! でそのまま即シェル実行</span></span>
<span class="line"><span class="prompt cy">&gt;</span> <span class="sx-key">!</span><span class="sx-flag">git status</span></span>
<span class="line"> </span>
<span class="line"><span class="sx-dim">  On branch main / nothing to commit…</span></span>
</CodeBlock>

これは **ターミナル版の入力テク** です。決まり切ったコマンドは `!` で、判断が要る作業は通常の指示で、と使い分けると往復が締まります。

## §3 Ctrl+S — 書きかけを退避する

長い指示を入力している最中に、急ぎの別の用が割り込んでくることがあります。打ちかけを消したくないけれど、先にそちらを片付けたい——そんなときに `Ctrl+S` が効きます。

書きかけのプロンプトをいったん退避させ、割り込みの操作を済ませたあとで書きかけに戻れます。せっかく組み立てた長文を捨てずに済むので、思考を途切れさせません。

<Callout>
  <b>これはターミナル版の小技です。</b> デスクトップアプリ版では入力欄の下書きが画面にそのまま残るため、明示的な退避操作は基本的に要りません。書きかけは入力欄に置いたまま、別の操作へ移れます。
</Callout>

## §4 Ctrl+G — 外部エディタで長文を書く

込み入った指示を一行ずつ入力欄で組み立てるのは骨が折れます。`Ctrl+G` を押すと、環境変数 `$EDITOR` で設定したお気に入りのエディタが開き、そこでプロンプトをじっくり書けます。

複数行の仕様や、構造を整えた長い依頼を書くときに重宝します。普段使いのエディタの操作感そのままで推敲でき、保存して閉じれば内容がそのまま送られます。

<CodeBlock title="zsh — エディタを指定しておく" badge="CLI">
<span class="line"><span class="sx-com"># 例: VS Code を $EDITOR にする（待機オプション付き）</span></span>
<span class="line"><span class="prompt">$</span> <span class="sx-key">export</span> EDITOR=<span class="sx-str">"code --wait"</span></span>
</CodeBlock>

これも **ターミナル版** の機能です。デスクトップ版は入力欄が広く複数行をそのまま扱えるため、外部エディタを呼ばなくても長文を組み立てられます。

## §5 Ctrl+B — 長い処理を裏に回す

時間のかかる処理を走らせている間、画面が占有されて別の作業ができないのはもったいない。`Ctrl+B` を押すと、その処理を <Term id="background-agent">バックグラウンドエージェント</Term> として裏に回し、手元では別の作業を続けられます。完了すると通知が来ます。

<ModeBlock mode="terminal">
ターミナル版では `Ctrl+B` で長い処理を背後へ送ります。前章までで触れた並列運用の、いちばん手軽な入口です。
</ModeBlock>

<ModeBlock mode="desktop">
デスクトップアプリ版では、複数セッションを並べて走らせること自体が標準の使い方です。裏に回すというより、<b>もう一つセッションを開いて並行させる</b>感覚で、進捗は画面上で見比べられます。
</ModeBlock>

## §6 --safe-mode — カスタムを無効化して高速起動

<Term id="hook">フック</Term> やプラグインを盛り込むと、起動時の初期化に時間がかかったり、設定同士が干渉したりすることがあります。そんなときは <Term id="safe-mode">--safe-mode</Term> です。

このオプションを付けて起動すると、カスタムの初期化をスキップして素早く立ち上がります。「サッと開きたいだけ」のときに速く、さらに **設定の切り分け確認** にも使えます。

<CodeBlock title="zsh — カスタム無効化で起動" badge="CLI">
<span class="line"><span class="sx-com"># フック/プラグインの初期化を飛ばして起動</span></span>
<span class="line"><span class="prompt">$</span> <span class="sx-key">claude</span> <span class="sx-flag">--safe-mode</span></span>
</CodeBlock>

<Callout>
  <b>切り分けの定石。</b> 「最近 Claude の挙動がおかしい」と感じたら、いちど <code class="inline">--safe-mode</code> で起動してみてください。<b>素の状態で直る</b>なら、原因は自分のフックやプラグイン側にある、と当たりがつきます。これは設定ファイルベースで効くため、起点はターミナルですが切り分けの考え方は両モード共通です。
</Callout>

## §7 小技は「引き出し」に入れておく

ここまでの小技は、すべてを暗記する必要はありません。「会話を汚さず聞きたい」「書きかけを残したい」「サッと起動したい」といった **困りごとが出たときに思い出せる** 程度で十分です。

<Callout type="warn">
  <b>最新版で確かめる癖を。</b> 本章のショートカットやオプションは版によって変わります。手元で効かないときは <code class="inline">/help</code> と公式の What's new を見るのが、いちばん確実な答え合わせです。
</Callout>

このサイトは Claude Code を **初級 → 初中級 → 中級 → 上級 → プロ** の五段で学べるように組んであります。各段は本の「部」と対応していて、上の段は下の段の上に積み上がります。つまり下を飛ばすと、上で足元がぐらつきます。

とはいえ、全員が第 0 部から読む必要はありません。この章のゴールは、**自分がどの段から読み始めればいいかを決められるようにする** こと。そのための地図を、まず一枚お見せします。

## §1 到達点マップ — 五段の階段

各レベルで「どこまで読むか」と「何ができるようになるか」を一覧にしました。気になる行から眺めてください。

| レベル | 読む範囲 | できるようになること |
| --- | --- | --- |
| **初級** | 第 0〜1 部 | インストールし、自然言語でコードを実装させる。安全に試し、失敗しても巻き戻せる |
| **初中級** | + 第 2 部 | <Term id="claude-md">CLAUDE.md</Term> と設定で Claude を自分仕様にし、ファイル編集から Git 操作まで任せる |
| **中級** | + 第 3 部 | <Term id="subagent">サブエージェント</Term>・<Term id="hook">フック</Term>・<Term id="mcp">MCP</Term> を使いこなす。ここで実務に投入できる |
| **上級** | + 第 4 部 | 複数エージェントを並列で走らせ、定期実行で自動化し、モデル選択でコストを最適化する |
| **プロ** | + 第 5 部 | CI に組み込み、<Term id="cli">CLI</Term> や SDK で部品化し、チームで共有運用しガバナンスを効かせる |

ポイントは **「+」** です。中級は「第 3 部だけ」ではなく「第 0〜3 部すべて」を指します。下の段の知識——巻き戻しで安全に試す感覚、CLAUDE.md で前提を伝える習慣——が土台になって、初めて上の段が活きます。

<Callout>
  <b>まずは初級の到達点だけ意識すれば十分です。</b> 「インストールして、日本語で頼んで、まずければ戻す」。この一周ができれば、もう Claude Code を使い始められます。残りの段は、必要になったときに登ればいい。
</Callout>

## §2 なぜ「サブエージェントを越えると中級」なのか

この本の難易度設計には、はっきりした基準が一本通っています。それが **「<Term id="subagent">サブエージェント</Term> を使いこなせたら中級者」** という線引きです。なぜこの一点を境にしたのか——理由は、Claude Code が実際に機能を増やしてきた **歴史の順番** と重なっているからです。

Claude Code は、おおよそ次の順で拡張されてきました。

- **<Term id="mcp">MCP</Term>** — 外部のデータやサービス(GitHub・Slack・DB など)を、Claude が自分の機能のように操作できる仕組み。
- **<Term id="subagent">サブエージェント</Term>** — 専門特化した別コンテキストの助手に作業を任せ、要約だけを受け取る仕組み。
- **<Term id="hook">フック</Term>** — Claude の動作の節目で、自分のスクリプトを自動で差し込む仕組み。
- **<Term id="skill">スキル</Term>** — 繰り返す手順を一つの入口にまとめ、`/名前` で呼び出せるようにする拡張。
- **<Term id="agent-team">エージェントチーム</Term>** — リード役が複数のセッションを束ね、サブタスクを割り振って結果をまとめる、複数エージェントの並列運用。

並べると分かります。前半(インストールして自然言語で頼む)は、Claude Code が登場した当初からできた **基礎の使い方** です。一方、サブエージェント以降は **「Claude にどう仕事をさせるか」を自分で設計する** 領域。ここから先は、ただ頼むだけでなく、役割を分け、節目に処理を挟み、手順を部品にまとめる——という一段違う発想が要ります。

<Callout>
  <b>サブエージェントが使えたら、あなたは中級者です。</b> 「専門の助手に、限られた権限で、一部の仕事を任せる」——この設計ができるようになると、Claude Code の使い方が一段変わります。だから本サイトは、ここを初級と中級の境目に置いています。
</Callout>

裏返すと、これは安心材料でもあります。<Term id="hook">フック</Term> や <Term id="mcp">MCP</Term>、<Term id="agent-team">エージェントチーム</Term> といった言葉に身構える必要は、今はありません。それらは中級・上級で順に登る段であって、初級のうちに理解していなくてまったく問題ないからです。

## §3 このサイトの歩き方

本編に入る前に、読みやすくするための仕掛けを二つだけ再確認します。どちらも前章までに触れていますが、ここからの章で頻繁に使うので、おさらいしておきましょう。

ひとつは **用語チップ** です。本文に点線の下線が引かれた語——たとえば <Term id="skill">スキル</Term> ——にカーソルを合わせる(スマートフォンならタップする)と、その場で短い説明がふわっと現れます。知らない言葉に出会ったら、まずそこを触ってみてください。別のページへ飛ばなくても、流れを止めずに意味を確かめられます。

もうひとつは **モードの切り替え** です。本サイトは、ターミナルから使う **ターミナル版** と、ウィンドウで操作する **デスクトップアプリ版** の二つで説明を進めます。右上のトグルを切り替えると、しずくが反対側へ移って、説明がその環境に合わせて入れ替わります。中身のモデルと能力はどちらも同じで、入り口だけが違います。選んだモードは記憶され、次に開いたときも維持されます。

<Callout type="warn">
  <b>段飛ばしは、足元に注意。</b> 急いで中級の <Term id="subagent">サブエージェント</Term> から読み始めても構いませんが、「巻き戻しで安全に試す」「<Term id="claude-md">CLAUDE.md</Term> で前提を伝える」といった土台が抜けていると、つまずきやすくなります。詰まったら、迷わず下の段に戻ってください。
</Callout>

## §4 さあ、入口を選ぶ

最後に、自分の入口を決めましょう。次の三つのうち、いちばん近いものを選んでください。

- **とにかく早く動かしたい人** — 第 1 部「インストールと初期設定」へ。手を動かしながら覚えるのが、いちばんの近道です。
- **概念から順に理解したい人** — このまま順番に。第 0 部の続きで全体像を掴んでから、第 1 部の実践へ進みます。
- **すでに基礎は分かっている人** — §1 の到達点マップで、自分がまだできないことが書かれた段を探し、その部から読み始めてください。

どの入口を選んでも、行き止まりはありません。分からない言葉は用語チップで確かめ、自分の環境はトグルで合わせる——その二つさえ覚えておけば、あとは地図のどこからでも歩き出せます。それでは、最初の段から始めましょう。

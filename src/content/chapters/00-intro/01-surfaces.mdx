前章で、Claude Code は「リポジトリごと任せる」エージェントだと掴みました。では、それをどこから使うのか。実は入り口は1つではありません。<Term id="cli">ターミナル</Term>からも、お使いのエディタからも、専用アプリからも、ブラウザからも使えます。

この「使う入り口」のことを <Term id="surface">サーフェス</Term> と呼びます。数が多くて最初は戸惑うかもしれませんが、この章でいちばん覚えてほしいのは、むしろ逆のこと——**入り口は5つあっても、中身は1つ** だという点です。

## §1 5つのサーフェス早わかり

まずは全体を俯瞰します。Claude Code を使う入り口は、大きく次の5つです。すべて同じ Claude のモデルと能力を使い、違うのは「どんな画面・操作で触るか」だけです。

| サーフェス | どんな入り口か | 向いている人 |
|---|---|---|
| **<Term id="cli">CLI</Term>**（ターミナル） | シェルから `claude` を起動して文字で対話。ファイル編集・コマンド実行・Git 操作まで全部こなせるフル機能版 | エンジニア、自動化したい人、Linux サーバーで使う人 |
| **VS Code 拡張** | 普段使う VS Code の中で動く。差分をエディタ内で確認、`@` でファイルを指定、編集の巻き戻しもできる | VS Code が仕事の中心の人 |
| **JetBrains 拡張** | IntelliJ・PyCharm・WebStorm などの中で動く。チャットと差分ビューを IDE に統合（別途 CLI の導入も要る） | JetBrains 系 IDE を使う人 |
| **デスクトップアプリ** | macOS / Windows 向けの専用アプリ。差分を視覚的に確認、複数の作業を並べて進められる（**Linux 非対応**） | 初心者、非エンジニア、PM、画面で確認したい人 |
| **Web・iOS** | ブラウザ（claude.ai/code）やスマホアプリから。手元に環境を作らなくても、長めの作業を投げて後で結果を見られる | 外出先で使う人、手元に無いリポジトリを触りたい人 |

<Callout>
  5つもありますが、全部を覚える必要はまったくありません。**自分が使う1つを選べば十分** です。本サイトでは、このうち2つ——ターミナル版とデスクトップアプリ版——にしぼって説明を進めます。理由はこのあとすぐ。
</Callout>

「これだけ入り口があると、サーフェスごとに使い方を覚え直さないといけないのでは……」と不安になるかもしれません。ここがこの章の山場です。結論から言うと、その心配は要りません。

## §2 いちばん大事なこと——土台は全サーフェス共通

サーフェスは入り口にすぎません。その奥にある **土台** ——あなたが作り込む設定やルール——は、**どの入り口から入っても同じように効きます**。

具体的には、次のような「土台」がそうです。

- **<Term id="claude-md">CLAUDE.md</Term>** — プロジェクトのルートに置く Markdown。コーディング規約や使うライブラリを書いておくと、毎回のセッション開始時に自動で読まれます。
- **設定（permissions など）** — 「どのファイルを読んでいいか」「どのコマンドを実行していいか」といった安全装置の取り決め。
- **<Term id="mcp">MCP</Term>** — GitHub や Slack、データベースなどの外部サービスを Claude に操作させる仕組み。

これらは一度プロジェクトに用意すれば、**ターミナルから使おうが、エディタから使おうが、アプリから使おうが、まったく同じように働きます**。CLAUDE.md に書いた規約は、どの入り口でも守られます。MCP で繋いだ GitHub は、どの入り口からでも使えます。

<Callout>
  <b>入り口が違うだけで、中身は1つ。</b> これは本サイト全体を貫く背骨です。後の章で学ぶ CLAUDE.md・設定・MCP・<strong>サブエージェント</strong>といった工夫は、特定のサーフェス専用の小ワザではありません。<b>一度作れば、どの入り口でも効く資産</b>になります。だから、サーフェス選びで悩む必要はないのです。
</Callout>

この事実があるおかげで、学ぶ側はとても楽になります。「ターミナル版で覚えたことが、デスクトップ版では通用しない」といったことが起きないからです。土台の作り方さえ身につければ、その知識はあなたが選んだどの入り口でもそのまま生きます。

## §3 本サイトは「ターミナル版」と「デスクトップアプリ版」の2モードで進めます

5つのサーフェスのうち、本サイトが扱うのは次の2つです。

- **ターミナル版**（<Term id="cli">CLI</Term>） — コマンドで操作するフル機能版。
- **デスクトップアプリ版**（<Term id="desktop">デスクトップアプリ</Term>） — ウィンドウで視覚的に操作する版。

この2つを選んだのは、**「文字で操作する代表」と「画面で操作する代表」** だからです。VS Code 拡張や JetBrains 拡張も結局はこのどちらかに近い使い心地ですし、何より §2 で見たとおり土台は共通なので、この2モードを押さえれば応用が利きます。

そして本サイトでは、説明そのものが選んだモードに合わせて切り替わります。いまあなたが見ているのは——

<ModeBlock mode="terminal">
### いまターミナル版を選んでいるあなたへ

**ターミナル版** の説明が表示されています。これ以降の章でも、コマンドを打ち込む手順を中心に解説が進みます。`claude` を起動して文字でやり取りするスタイルです。
</ModeBlock>

<ModeBlock mode="desktop">
### いまデスクトップ版を選んでいるあなたへ

**デスクトップアプリ版** の説明が表示されています。これ以降の章でも、アプリの画面を操作する手順を中心に解説が進みます。ウィンドウでフォルダーを開いてやり取りするスタイルです。
</ModeBlock>

いま表示されている説明が、あなたが選んでいるモードです。試しに切り替えてみましょう。

<ModeText mode="terminal">ページ右上のトグルを **デスクトップ** 側に動かすと、しずくが反対側へとろりと移って、上の囲みの中身も、このあとの章の説明も、すべてデスクトップアプリ版に入れ替わります。</ModeText>
<ModeText mode="desktop">ページ右上のトグルを **ターミナル** 側に動かすと、しずくが反対側へとろりと移って、上の囲みの中身も、このあとの章の説明も、すべてターミナル版に入れ替わります。</ModeText>
選んだモードは記憶されるので、次にサイトを開いたときも同じモードのまま読み進められます。

<Callout>
  まだ自分がどちらを使うか決めていなくても大丈夫です。とりあえず気になる方を選んで読み進め、合わなければいつでも右上で切り替えられます。<b>どちらで学んでも、土台の知識は共通</b>——これがこの章の結論です。
</Callout>
